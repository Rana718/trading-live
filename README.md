# 24時間自動配信システム — 使い方ガイド

## 起動方法

以下のコマンドを1回だけ実行してください。

```
uv run python start.py
```

ブラウザが自動で開きます。開かない場合は [http://localhost:5000](http://localhost:5000) にアクセスしてください。

---

## 画面の使い方

### 1. 配信の開始・停止

画面上部に **▶ 配信開始** と **■ 配信停止** ボタンがあります。

- **配信開始** を押すと YouTube Live への配信が始まります
- **配信停止** を押すと配信が完全に止まります
- ボタン横の緑のランプが点灯しているときは配信中です

---

### 2. 設定の保存

設定を変更したら、必ず画面上部の **💾 設定を保存** ボタンを押してください。  
保存しないと変更は反映されません。

---

### 3. 各設定項目

#### YouTube 配信設定

| 項目 | 説明 |
|------|------|
| ストリームキー | YouTube Studio から取得したキーを貼り付けてください。右の 👁 アイコンで表示/非表示を切り替えられます |
| データソース | 価格の取得元。通常は **CoinGecko** のままで問題ありません |

**ストリームキーの取得方法:**
1. [YouTube Studio](https://studio.youtube.com) を開く
2. 左メニューの「配信」をクリック
3. 「ストリームキー」をコピーして貼り付ける

---

#### 更新間隔

| 項目 | 説明 |
|------|------|
| チャート更新（秒） | チャートを何秒ごとに更新するか。初期値: 60秒 |
| ニュース読み上げ間隔（秒） | ニュースを何秒ごとに読み上げるか。初期値: 300秒（5分） |
| 銘柄切り替え間隔（秒） | 複数銘柄を設定した場合の切り替え間隔。初期値: 300秒 |

---

#### 音声読み上げ設定

| 項目 | 説明 |
|------|------|
| 音声 | 読み上げの声を選びます。Nanami（女性）または Keita（男性） |
| 速度 | 読み上げ速度。`+10%` で速く、`-10%` で遅くなります |
| 音量 | 読み上げ音量。`+20%` で大きく、`-20%` で小さくなります |

---

#### 銘柄リスト

配信するチャートの銘柄を設定します。

| 項目 | 説明 |
|------|------|
| CoinGecko ID | 銘柄の識別子。例: `ripple`、`bitcoin`、`ethereum` |
| 表示名 | 画面に表示される名前。例: `XRP/JPY`、`BTC/JPY` |
| 通貨ペア | `jpy`（日本円）または `usd`（米ドル） |
| Binanceシンボル | Binanceを使う場合のみ入力。通常は空白でOK |

**＋ 銘柄を追加** ボタンで銘柄を増やせます。  
右上の ✕ で削除できます。

---

#### ニュースソース

読み上げるニュースの取得元を設定します。

| 項目 | 説明 |
|------|------|
| タイプ | `RSS` を選んでください（通常はRSSで問題ありません） |
| URL | RSSフィードのURL |

**＋ ニュースソースを追加** ボタンで追加できます。

---

## BGM について

`music.mp3` というファイルをこのフォルダに置くと、配信中に自動でBGMとして流れます。  
読み上げ音声はBGMより大きく聞こえるよう自動調整されます。

---

## よくある質問

**Q. 配信が始まらない**  
→ ストリームキーが正しく入力されているか確認してください。

**Q. 音声が聞こえない**  
→ インターネット接続を確認してください。音声生成にはネット接続が必要です。

**Q. ニュースが更新されない**  
→ ニュースソースのURLが正しいか確認してください。

**Q. チャートが表示されない**  
→ しばらく待ってください。起動直後はデータ取得に数十秒かかる場合があります。
